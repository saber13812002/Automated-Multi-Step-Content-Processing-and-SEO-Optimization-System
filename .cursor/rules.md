# Project Rules

- از ساختار موجود پوشه‌ها پیروی کنید و کدهای مرتبط با ChromaDB را در دایرکتوری `export-sql-chromadb` نگه دارید.
- همیشه قبل از اتصال به ChromaDB یا Redis، متغیرهای محیطی را از `.env` یا متغیرهای سیستم بارگذاری کنید.
- لاگ‌ها باید ساختار JSON داشته باشند تا در سیستم‌های مانیتورینگ به راحتی پردازش شوند.
- endpointهای HTTP باید پایدار و قابل تست با ابزارهای ساده مانند `curl` و `httpie` باشند.
- در صورت اضافه کردن وابستگی جدید، آن را در `web_service/requirements.txt` یا فایل requirements مربوطه بنویسید.
- پس از ایجاد یا تغییر سرویس‌ها، مسیر اجرای Docker (`Dockerfile`, `docker-compose.yml`) را به‌روزرسانی کنید.
- مستندات جدید را به `export-sql-chromadb/README.md` اضافه کنید تا روند راه‌اندازی روشن بماند.
